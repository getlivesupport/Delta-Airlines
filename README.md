## 📱 Mobile Performance Rules

All pages built to these standards:
- **No external CSS files** — all styles inline in `<style>`
- **No JS libraries** — vanilla JS only, IIFE wrapped
- **Passive scroll listeners** — prevents janky scrolling on mobile
- **Font preconnect** — reduces render-blocking time
- **Mobile sticky CTA** — fixed bottom bar, dismissible
- **1-column grid** on all screens below 480px
- **clamp() font sizes** — fluid typography, no zoom issues
- **prefers-reduced-motion** — accessibility compliance
- **focus-visible** — keyboard navigation support
- **Target Lighthouse:** 95+ Performance, 100 Accessibility, 100 SEO, 100 Best Practices

## 🔗 Internal Linking Rules

**All links MUST use full GitHub Pages path:**
```html

❌ WRONG:     href="/flight-cancellation/"
❌ WRONG:     href="flight-cancellation/"
❌ WRONG:     href="https://getlivesupport.github.io/flight-cancellation/"
```

GitHub Pages serves from a subpath — root-relative links without
the repo name will 404 on every cluster page.

## ⚠️ What NOT to Do

- Never claim official affiliation with Delta Air Lines, Inc.
- Never remove the legal disclaimer section from any page
- Never block AI crawlers in robots.txt (GPTBot, Claude-Web, PerplexityBot)
- Never use the same meta description on two pages
- Never duplicate H1 tags across pages
- Never let `dateModified` go stale longer than 60 days
- Never add fake urgency ("Only 2 seats left!") — Google's spam classifier flags this
